工作区和暂存区

  • 工作区：
    在创建了 Git 仓库的文件夹，就是一个工作区。

  • 版本库：
    在 Git 仓库的文件里，有一个 .git 的文件夹，这个不算是工作区，而是 Git 的版本库。
    版本库中有称为 Stage 的暂存区，还有 Git 自动创建的分支 master。指向 master 的一个指针叫做 HEAD。

git add命令实际上就是把要提交的所有修改放到暂存区，然后，执行git commit就可以一次性把暂存区的所有修改提交到分支。

远程仓库

通过 SSH 连接远程仓库

首先创建 SSH Key。

在主目录的 .ssh 中可以看到 id_rsa 和 id_rsa.pub 两个文件，这就是秘钥对。前者是私钥，不能泄露，而后者是公钥，可以放心地告诉他人。

接着登录 GitHub， 打开 Account settings -> SSH Keys，点击 Add SSH Key，填上你喜欢的名字，在 Key 文本框中把公钥的内容粘贴进去。随后点击 Add Key。

Git 自定义

Git 上色
忽略特殊文件

在 Git 工作区根目录下创建一个 .gitignore 文件，把需要忽略的文件名填进去，Git 就会忽略这些文档。
原则：

  1. 忽略操作系统自动生成的文件。
  2. 忽略编译生成的中间文件、可执行文件等。
  3. 忽略你自己的带有敏感信息的配置文件。

把 .gitignore 文件提交到 Git，就会自动生效。
在被设定为被忽略文件时，若想要提交，可以使用：
$ git add -f <filename>
或者找出是哪个规则出现了问题：
git check-ignore -v <filename>
想要在 .gitignore 的规则下不忽略某些文件，可以使用：
! + <filename>

配置 Git 别名

下列命令就是用 st 代替 status：
$ git config --global alias.st ’status’
其中， --global 参数是全局参数，这意味着这些命令在 这台电脑 下的所有 Git 仓库都适用。
配制文件实际上存放在 .git/config 文件中。而用户配置的 Git 配制文件则在 用户主目录 下的隐藏文件 .gitconfig 中。这意味着配置别名也可以直接修改这个文件。
