基于 Hostwinds 的 Wireguard 搭建和 V2ray 一键安装脚本搭建

Written by: GummyBear

本文记录了通过 Wireguard 和 V2ray 两种途径，在 Hostwinds 的 VPS 主机上实现科学上网的坎坷过程。笔者的初心是搭建 Wireguard 来实现科学上网，但是无奈，Wireguard 的客户端需要科学上网才能获取，于是通过了别人写好的脚本搭建好了 V2ray 的环境后，才成功获得 Wireguard 的客户端，从而实现了 Wireguard 和 V2ray 两种途径的科学上网的实现。

Hostwinds VPS 主机的购买

1、通过 Hostwinds.com 访问 Hostwinds 官网，如图，选择 VPS -> Linux Unmanaged，随后选择最便宜的（一个月的），进行购买。

2、接下来按照要求进行注册。

3、选择 VPS 地点和操作系统。题主选择的是西雅图 + CentOS7。随后可以使用支付宝进行支付，支付完成后，如图所示。

4、在注册时填写的邮箱中会收到一封邮件，里面记录了你所购买的 VPS 的各种参数。

至此，购买 VPS 的准备工作完成。

使用 XShell 通过 SSH 远程连接 Hostwinds VPS

1、打开 XShell ，点击 “文件” -> “新建” 。

2、随便给这个会话取个名字，然后把邮件中收到的 VPX 的 Main IP 填上去，SSH 端口也填上去，然后点击连接。

然后就不太顺利了，在这里题主遇到了第一个问题。如图所示：

XShell 出现了 Connection failed 的报错，通过网上大量查阅信息之后，发现可能是因为 Hostwinds 的一些 ip 地址被墙了导致的。于是通过 该链接 进行 ip 的可用性测试。结果如图所示：

结果很明显，国外可用而国内不可用，属于是被墙了（悲）。通过查询大量资料后，发现了 Hostwinds 对中国的用户提供了免费更换 IP 的服务。只需：

  1. 登陆你的 Hostwinds 账号。
  2. 右上角点击自己的头像。
  3. 看到自己的 active 的机器，选择右边的 manage。
  4. 滑动滚轮，在页面下方点击 Click Here to Manage This Server。
  5. 在这一行中选择 Manage IP's。
  6. 在这里选择 Fix ISP Block。
  7. 点击确认后，只需等待几分钟，就可以看到自己的全新 IP 了。还没完，需要在上面点击 Actions -> Reinstall。

至此，IP 地址的更换已经成功。接下来通过测试网页进行测试，结果是显示可用的。（如果不可用的话，可能需要换多几次）再次通过 XShell 尝试连接。

出现如图所示的现象，那就意味着远程连接成功。

兴高采烈地点击 启动 ~

本该想着美好的科学生活就要开始，但是，网络却直接炸了！！！！
不但小企鹅瞬间倒地，连百度也上不了了。

回到 Wireguard 的客户端界面，我发现接受的数据只有 90B 。我百思不得其解，上网查询了大量资料，尝试过更改链接的监听端口，也考虑过可能时内网冲突，尝试使用过不同的内网 IP ，都没有成功。就在我一筹莫展的时候，网上一个冷清清的帖子提出的解决方法让我的 Wireguard 如获新生。

原来是 Linux VPS 的 IP Forwarding 没有被启用，通过在服务端执行如下命令：
sysctl -w net.ipv4.ip_forward=1
（永久性修改： /etc/sysctl.conf 中设置 net.ipv4.ip_forward = 1 ）

随后重启测试 Wireguard，成功！
